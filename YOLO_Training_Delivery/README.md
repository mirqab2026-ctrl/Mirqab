# حزمة تدريب نماذج YOLO — مشروع مرقاب

نسخة نظيفة ومنظّمة من جميع ملفات تدريب نماذج الرؤية الحاسوبية المستخدمة في مشروع **مرقاب** (بوابة أمنية ذكية)، جاهزة للتسليم للمشرف واللجنة.

تغطي الحزمة نموذجين مدرّبين بتقنية YOLOv8:

1. **كاشف اللوحات** (`license_plate_detector.pt`) — يحدد موقع لوحة السيارة في الصورة.
2. **قارئ الأحرف OCR** (`ocr_chars_detector.pt`) — يقرأ محتوى اللوحة (أحرف وأرقام، عربي + إنجليزي).

يعمل النموذجان معاً كخط متكامل: كشف اللوحة ← قصّها ← قراءة أحرفها ← ترتيبها ← إخراج النص النهائي.

| المقياس | القيمة |

| عدد الحقب (Epochs) | 50 |
| حجم الصورة | 640×640 |

| الاسترجاع Recall | 0.908 |
| **mAP@0.5** | **0.948** |

### نموذج قراءة الأحرف (OCR)

| المقياس | القيمة |
|---|---|
| النموذج الأساس | YOLOv8n |
| عدد الحقب (Epochs) | 80 (توقّف مبكر عند 67) |
| حجم الصورة | 640×640 |
| عدد الفئات | 27 فئة (17 حرف + 10 أرقام) |
| الدقة Precision | 0.963 |
| الاسترجاع Recall | 0.954 |
| **mAP@0.5** | **0.986** |
| mAP@0.5:0.95 | 0.756 |

> ملاحظة: كل فئة حرفية تمثّل النسختين العربية والإنجليزية لنفس الحرف وفق خريطة الأحرف `character_map.json`.

## كيف يعمل الخط المتكامل

```
صورة سيارة
   ↓
1. كاشف اللوحة (license_plate_detector.pt) → تحديد مربع اللوحة
   ↓
2. قصّ اللوحة من الصورة
   ↓
3. قارئ الأحرف (ocr_chars_detector.pt) → كشف الأحرف ومواقعها
   ↓
4. ترتيب الأحرف حسب الموقع (يسار→يمين)
   ↓
5. التحويل لأحرف فعلية عبر character_map.json
   ↓
النص النهائي للوحة
```

## كيفية إعادة الاستخدام / التشغيل

تشغيل الخط المتكامل على صورة (يتطلب Python مع `ultralytics` و`opencv-python`):

```python
from plate_ocr import PlateOCR

ocr = PlateOCR(
    plate_model="../01_النماذج_المدربة/license_plate_detector.pt",
    char_model="../01_النماذج_المدربة/ocr_chars_detector.pt",
    char_map="../04_الداتاست/character_map.json",
)
text = ocr.read("test_images/2.jpeg")
print(text)
```

لإعادة التدريب: افتح الدفاتر في `03_كود_التدريب/` على Google Colab مع تفعيل GPU، وارفع الداتاست من `04_الداتاست/`.

## ملاحظات حول تنظيف الحزمة

أُعدّت هذه النسخة من مجلد العمل الأصلي مع المعالجات التالية لضمان النظافة والسلامة:

- **إزالة التكرار:** المجلد الأصلي كان يحتوي على نسخة داخلية مكررة بالكامل (`yoloOD/yoloOD`)؛ تم الاعتماد على نسخة واحدة فقط.
- **إصلاح الداتاست:** الملف الأصلي `merged_license_plate_dataset.zip` كان **تالفاً/ناقصاً (25 ميجا)**؛ تم استبعاده وإعادة دمج الأجزاء `merged_part_aa…ad` في ملف واحد صحيح ومُتحقَّق منه: `merged_license_plate_dataset_full.zip` (94 ميجا، 5855 ملف).
- **تنظيم المخرجات:** فُكّت ملفات نتائج التدريب المضغوطة وصُنّفت ضمن مجلدات واضحة، مع فصل النماذج والكود والداتاست ونتائج التدريب وخط الاستدلال.
