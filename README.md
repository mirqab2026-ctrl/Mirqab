# مرقاب · Mirqab
### نظام ذكي للتعرّف على لوحات المركبات والوجوه عند البوابات

لوحة فحمي + ذهبي · واجهة عربية · يعمل دون اتصال (Air-gapped)

</div>

---

## نظرة عامة
«مرقاب» نظام تحكّم بالبوابات يدمج رؤية الحاسب والتعلّم الآلي لاتخاذ قرار دخول موثوق:

- كشف لوحة المركبة وقراءتها (YOLOv8 + OCR) بدعم المعيار السعودي.
- التعرّف على وجه السائق (InsightFace buffalo_s — ArcFace 512-d عبر ONNX، خفيف على CPU)
  بعتبة تطابق **موحّدة** ثابتة في `backend/config.py`.
- دمج القرار عبر أربعة أوضاع تشغيل (صارم / متوازن / عرض / طوارئ).
- لوحة معلومات، بحث شامل، تسجيل، مركز تدريب، تقارير، ومركز تحكّم.

## المتطلبات
- Python 3.10 أو أحدث.
- ويندوز (يعمل أيضاً على لينكس/ماك).

## التثبيت والتشغيل
1. شغّل `setup.bat` مرة واحدة لتثبيت المتطلبات وتنزيل نماذج الوجه
   (أو يدوياً: `pip install -r requirements.txt` ثم `python scripts/setup_face_models.py`).
2. إن كانت لديك ترميزات وجه قديمة (dlib 128-d): `python scripts/migrate_face_encodings.py`.
3. شغّل `run.bat` لفتح الواجهة (أو: `streamlit run frontend/app.py`).

> لمعايرة عتبة الوجه الموحّدة على بياناتك: `python scripts/calibrate_face_threshold.py`
> ثم ثبّت القيمة في `backend/config.py`.

### بيانات الدخول
تُقرأ من متغيّرات البيئة لأسباب أمنية، وتُحمّل من ملف محلي **غير مرفوع للمستودع**:

1. انسخ `credentials.example.bat` إلى `credentials.local.bat`.
2. غيّر القيم بداخله:

```bat
set GATE_USERNAME=admin
set GATE_PASSWORD=كلمة-مرور-قوية
```

> `credentials.local.bat` محمي بـ `.gitignore` فلا تُرفع كلمة المرور أبداً.
> إن لم يوجد الملف، يعمل النظام بـ `admin/admin` مؤقتاً مع عرض تحذير في واجهة الدخول.

## تنزيل النماذج (غير مضمّنة في المستودع)
النماذج المدرّبة كبيرة الحجم (~42 ميجا) ولا تُرفع للمستودع للحفاظ على خفّته. للحصول عليها:

- **نماذج الوجه** (`det_500m.onnx` و `w600k_mbf.onnx`): تُنزَّل تلقائيًا عبر
  `python scripts/setup_face_models.py`.
- **نماذج اللوحة والأحرف** (`license_plate_detector.pt` و `ocr_chars_detector.pt`)
  و `character_map.json`: حمّل حزمة المشروع الكاملة من الرابط أدناه، وانسخ مجلد `models/`
  إلى جذر المشروع.

> 📦 **تنزيل المشروع الكامل / النماذج (Google Drive):**
> `ضع رابط Google Drive هنا بعد الرفع`

البنية المتوقّعة لمجلد النماذج:
```
models/
├─ license_plate_detector.pt
├─ ocr_chars_detector.pt
├─ character_map.json
└─ face/
   ├─ det_500m.onnx
   └─ w600k_mbf.onnx
```

## بنية المشروع
```
Mirqab_System/
├─ frontend/        واجهة Streamlit (app.py + الصفحات + الثيم)
├─ backend/         المنطق: قاعدة البيانات، اللوحات، الوجه، القرار، التحقق
├─ models/          نماذج الكشف والقراءة (.pt) + خريطة الأحرف
├─ data/            قاعدة البيانات gate.db + الصور والسجلات
├─ assets/fonts/    خط Saudi الرسمي + شعار مرقاب
├─ scripts/         أدوات تشغيلية (البذر، القياس، التقييم)
├─ .streamlit/      إعدادات Streamlit
├─ run.bat          تشغيل الواجهة
└─ setup.bat        تثبيت المتطلبات
```

## نتائج التقييم المعتمدة (2026-06-06)
قياس فعلي موثّق — المراجع: `scripts/run_full_benchmark.py` و `scripts/benchmark_face.py`:

| النظام | المؤشر | النتيجة |
|---|---|---|
| كاشف اللوحات (YOLOv8s) | معدل الكشف (26 صورة) | **100%** |
| قارئ الأحرف (YOLOv8n) | قراءة صالحة / مثالية | **100% / 84.6%** |
| النظام الكامل E2E | نجاح كامل | **100%** |
| الوجه (buffalo_s · 512-d) | Rank-1 على 310 مجسّ / 62 شخصاً | **100%** |
| الوجه | قبول دخيل FAR / رفض خاطئ FRR | **0.0% / 0.0%** |
| الوجه | زمن كشف+تشفير (CPU) | **33ms متوسط · 89ms p95** |

> عتبة الوجه الموحّدة: **0.58 cosine** — معايرة على 1230 زوجاً حقيقياً و4005 أزواج دخيلة
> (`scripts/calibrate_face_threshold.py`)، مع تنظيف معرض الوجوه من البذر التركيبي
> المتشابه (`scripts/clean_synthetic_faces.py`).
